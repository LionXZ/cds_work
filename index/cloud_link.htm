云连接(Cloud Link)
目前首云已经有多个客户希望通过首云连接到Amazon Web Service(AWS)的实际案例，客户在AWS 使用Direct Connect产品，然后通过首云提供的物理链路和设备打通首云和AWS之间的网络，实现首云VDC和AWS VPC私网互通，完成业务组网。首云的云连接服务是通往AWS资源的优选路径。当您有跨云业务需求时，您可以选择由首云提供的云连接整体解决方案。
核心优势
首云做为同时拥有公有云和IDC的全球云服务提供商，有丰富的多云互联经验，积极帮助客户设计跨云部署方案，并提供方案所需的设备和链路资源。
首云做为AWS的APN合作伙伴，对AWS产品有更深入的了解，AWS专属服务团队累积了大量的落地经验，能帮助客户更好的使用AWS的产品和服务。
首云提供完善的对接流程和售后服务，客户在使用首云的云连接服务中，首云提供7*24小时人工服务，为客户的业务保驾护航。
成功案例
  • 首云-AWS业务互通解决方案
    首云-AWS业务互通解决方案
    AWS：
    客户在AWS的新加坡、达拉斯和法兰克福等多个可用区部署了业务，并且该业务同样需要在首云中国大陆和中国香港部署，两个云厂商多个区域的业务需要连通。
    首云：
    客户将中国大陆和中国香港的业务部署在首云，需要实现这些业务与部署在AWS海外多个地区的业务互联互通。首云提供连接设备和链路，利用AWS的Direct Connect 服务为客户完成了业务互通。
    特点
    保证客户业务连通性
    提升业务稳定性
  • 首云多云互联解决方案
    首云多云互联解决方案
    AWS和GCP：
    客户目前在AWS和GCP的多个区域进行了业务部署，属于典型的多云部署模式。该客户同样在首云也有业务部署，客户希望将首云、AWS和GCP多个云厂商的业务互联互通。
    首云：
    其中，首云和AWS的互联是通过使用AWS的Direct Connect 服务，结合首云提供的设备和链路将两端网络互通。在业务部署的过程中，首云无论是设备还是链路都是多活模式，保证客户对业务所需的SLA。
    特点
    多云部署
    多活容灾
    专业的技术服务团队
